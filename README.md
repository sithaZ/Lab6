## Objective

The purpose of this lab is to learn how to perform load testing using Gatling. The lab demonstrates how to create simulations, configure protocols, use feeders, apply assertions, and analyze performance reports generated by Gatling.

---

## Tools Used

* Java JDK 17
* Apache Maven
* Gatling 3.14
* Visual Studio Code
* GitHub

---

## Exercise 1: Open Model Simulation

### Features Implemented

* HTTP Protocol Configuration
* Scenario Definition
* CSV Feeder
* Status Code Checks
* Assertions

### Injection Profile

```java
rampUsers(20).during(30)
```

This Open Model gradually increases users from 0 to 20 over 30 seconds.

### Result

* Total Requests: 40
* Successful Requests: 40
* Failed Requests: 0
* Success Rate: 100%

---

## Exercise 2: Closed Model Simulation

### Injection Profile

```java
constantConcurrentUsers(10).during(30)
```

This Closed Model maintains 10 concurrent active users throughout the simulation.

### Result

* Total Requests: 273
* Successful Requests: 273
* Failed Requests: 0
* Success Rate: 100%
* Mean Response Time: 98 ms

---

## Assertions

The following assertions were used:

```java
global().responseTime().mean().lt(1000)
global().successfulRequests().percent().gt(95.0)
```

All assertions passed successfully.

---

## HTML Report Analysis

The Gatling HTML report was used to analyze:

* Active Users
* Requests per Second
* Response Time Distribution
* Response Time Percentiles
* Success and Failure Rates

The application handled the simulated load successfully with no failed requests.

---

## Conclusion

This lab demonstrated how to use Gatling for load testing. Open Model and Closed Model simulations were created and executed successfully. Feeders, checks, and assertions were implemented to validate application performance. The generated HTML reports provided useful insights into response times, throughput, and user load. Gatling can be integrated into CI/CD pipelines to automatically validate performance requirements before deployment.
